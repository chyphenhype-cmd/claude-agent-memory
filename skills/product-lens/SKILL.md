## The Product Lens

Every feature, every screen, every decision should pass through three filters:

### 1. User Filter — "Would a real person love this?"
- Put yourself in the shoes of someone actually using this tool
- Walk through the flow step by step as a non-technical user would
- Ask: Is this confusing? Is there unnecessary friction? What would they expect to happen?
- The best features feel obvious AFTER you use them — no manual needed
- If you need to explain how a feature works, the UX is wrong

### 2. Business Filter — "Who pays for this and why?"
- Every feature is an investment — what's the return?
- Can this feature be a selling point? A differentiator from competitors?
- Would someone switch from their current tool because of this?
- What's the simplest version that delivers 80% of the value?
- Price on value delivered to the customer, not cost to build

### 3. Build Filter — "Are we building the right thing the right way?"
- Is this solving a real problem or a hypothetical one?
- What's the fastest path to something users can touch and react to?
- Can we validate demand before building the full thing?
- Are we over-engineering or under-engineering for the stage we're at?

## When Building Features, Always Surface:
- **Who benefits**: "This helps [user type] because [reason]"
- **Market angle**: "Other [industry] tools charge $X for this" or "Nobody does this well yet"
- **MVP scope**: "The minimum version that proves value is [X]"
- **Revenue connection**: "This could become a paid feature because [reason]"

## Red Flags to Call Out:
- Building for ourselves without considering if others have this problem
- Adding complexity that doesn't map to user value
- Skipping the "would someone pay for this?" question
- Beautiful code that solves no one's problem
- Features that require explanation instead of being intuitive

## Product Patterns (grows over time):
- Vertical SaaS (niche tools) can charge 3-5x horizontal tools
- Solve your own problem first, then package for others like you
- Domain expertise encoded into software IS the moat
- The best demo is "let me show you how I use it every day"
- Free tools with paid premium features convert better than paywalls
